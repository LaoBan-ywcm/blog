移动Web手册 (浏览器)

发表于 2015-07-21

手机上有4种浏览器类型：内置浏览器、可下载浏览器、代理浏览器、以及WebView。但是也会有重叠。

内置浏览器：
  • IOS : Safari
  • Windows Phone: IE
  • 安卓：安卓WebKit 或 Chrome
  • 黑莓：黑莓WebKit
  • Firefox OS：火狐

无法单独升级浏览器，必须升级系统，这造成开发缓慢。

可下载浏览器：

ios上不允许安装其他的渲染引擎。

可下载浏览器:Firefox, Opera, UC, QQ, 360, etc

代理浏览器：

代理浏览器的渲染引擎能够解析和执行HTML、CSS还有JavaScript，但并不是运行在设备上，而是在远程服务器上。

三种比较重要的代理浏览器：Opera Mini、UC Mini、Nokia Xpress。

优点是：轻量级，可以跑在低规格的手机上。
缺点是：没有客户端交互。

WebView：

WebView是留给原生应用的一个操作系统浏览接口。

苹果不允许在ios设备上安装其他渲染引擎。因此，其他浏览器要想在IOS上安装就必须用苹果的WebView。

渲染引擎

不使用WebKit的浏览器：

  1. IE Mobile 使用 Trident
  2. Opera Mini 使用 Presto，但是最终会换成 Blink
  3. Chrome 浏览器使用 Blink
  4. Firefox Mobile 和 Firefox OS 使用 Gecko
  5. UC Mini、Nokia Xpress 还有 Jolla 的 Sailfish OS 上的内置浏览器也用的是 Gecko
安卓

安卓内置浏览器情况复杂：

  • 旧的内置浏览器—-安卓WebKit，正逐渐被Chrome替代
  • 安卓Webkit和Chrome有不同的衍生版

为了实现差异化，设备厂商对内置浏览器做了修改。

Google Chrome 是谷歌服务的一部分，因此它在所有支持谷歌服务的安卓4.3+设备上可用。有一个很关键但经常被误解的观点—-Google Chrome 并不一定是设备的内置浏览器。

安卓WebKit并不是Chrome：它有完全分离的代码库，里面包含了完全分离的漏洞。渲染引擎是WebKit，而不是Blink。最新的安卓WebKit的版本是4.3，谷歌不再更新和支持它，安卓Webkit在安卓4.4中不再可用，谷歌打算用Chrome来代替它。

在2013年，谷歌将Chrome从WebKit分离出来，并创造了自己的渲染引擎Blink。

Blink是为HTML和CSS设计的渲染引擎，通常与V8 JavaScript引擎配合使用。Blink与人人都可下载和修改的谷歌开源浏览器Chromium紧密地集成在一起。Chromium是为Windows、Mac、Linux、ChromeOS和安卓设计的。Google Chrome是谷歌自己对Chromium的一个实现。

从Galaxy S4(2013发布) 以来，三星开始使用他自己开发的基于Chromium的一款浏览器作为内置浏览器，称为三星Chrome。

谷歌强制所有使用谷歌服务的设备使用Google Chrome。所以很多手机，除了自己的内置浏览器，也都安装了Google Chrome。

如果navigator.userAgent包含单词Chrome，则是Chrome（不一定是Google Chrome），不包含，就是安卓WebKit。
